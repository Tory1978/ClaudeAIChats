# ClaudeAIChats
Шаг 1 — Экспортировать чаты из Claude.ai
Открой claude.ai
Нажми на своё имя / аватар в левом нижнем углу
Выбери Settings (Настройки)
Найди раздел "Privacy"
Нажми кнопку "Export data"
На почту придёт письмо со ссылкой — скачай архив .zip
Внутри архива будет папка с файлами .json — это твои чаты.

Шаг 2 - Загрузить на GitHub
Открыть закладку Code 
Убедись, что ты находишься в ветке archive (кнопка должна показывать archive)
Если нет — нажми на кнопку с названием ветки и выбери archive из списка
Нажми "Add file" → "Upload files"
Нажми "Commit changes"

Шаг 5 — Удалить файлы из ветки (опционально)
Переключись на ветку 
Найди загруженные файлы чатов
Открой файл → нажми иконку корзины (🗑) справа вверху
Нажми "Commit changes"
========================================
К сожалению, на данный момент экспорта Projects из Claude.ai не существует — эта функция не предусмотрена.
Но можно сохранить всё вручную. Вот что обычно хранится в проекте и как это сохранить:

1. Инструкции проекта (Project instructions)

Открой проект в Claude.ai
Скопируй текст инструкций
Создай файл instructions.md в GitHub и вставь туда текст

2. Чаты внутри проекта

Они войдут в общий экспорт через "Export data", который ты уже делала ✅

3. Загруженные файлы (если есть)

Скачай их вручную из проекта
Загрузи в GitHub

Где хранить в GitHub?
Можно создать такую структуру в ветке archive:
archive/
  chats/
    2026-06/
  projects/
    название-проекта/
      instructions.md
      файлы/


=================================
Артефакты из Claude.ai (код, документы, SVG и т.д.) не экспортируются автоматически — их тоже нужно сохранять вручную.

Как сохранить артефакт:

Открой артефакт в чате
Нажми кнопку "Copy" (скопировать содержимое)
Создай файл в GitHub с нужным расширением:

код на Python → script.py
веб-страница → index.html
документ → document.md


Вставь содержимое и сохрани


Где хранить в GitHub?
Можно дополнить твою структуру в ветке archive:
archive/
  chats/
    2026-06/
  projects/
    название-проекта/
  artifacts/
    2026-06/
      script.py
      document.md

Совет: называй файлы понятно, например 2026-06-11-анализ-данных.py — чтобы потом было легко найти нужный артефакт.
